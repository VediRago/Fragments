# Luck vs Outcome

This file clarifies how unlikely events should be understood inside Fragments.

What looks obvious from the author or system level may not be obvious to a reader, so this rule exists to prevent the story from accidentally making rare events feel like chosen destiny, divine guidance, or random convenience.

This is an internal logic note, not a rule the story needs to explain directly.

## Core Rule

To a fragment, rare events feel like luck.

To Qalm, they are not random in the same way. Qalm does not choose lucky outcomes, but because it preserves all possible outcomes, even extremely unlikely paths exist somewhere.

This means coincidences can feel miraculous from inside a life while still being structurally possible inside the larger system.

## Important Distinction

Qalm does not arrange events for a preferred result.

Qalm does not reward, punish, guide, or select special fragments.

Qalm preserves total possibility.

So an event can be:

- emotionally miraculous from inside the character's life
- statistically inevitable somewhere across total possibility
- not personally chosen by the system
- not random from the system's full outcome scale

This rule protects the story from treating rare events as proof that Qalm wants something.

## Working Phrase

Not chosen.

Not random.

Permitted by total possibility.

## Reader-Facing Use

The story should not explain this rule directly.

Readers may experience certain events as coincidence, miracle, fate, or luck. That is acceptable from inside the story's emotional perspective.

The reference rule exists only to keep the author logic stable.
