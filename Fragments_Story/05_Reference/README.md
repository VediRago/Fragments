# Story Reference

This folder contains deeper reference material for **Fragments**.

These files are not the main public reading route. They exist to keep the story structure, bridges, cosmology, Unity logic, perception rules, and emotional patterns consistent while the project develops.

## Use This Folder For

- bridge logic between scenes, characters, and motifs
- emotional maps across acts
- misdirection and reflection patterns
- deep cosmology notes
- Unity explanation and limits
- internal story principles that should not interrupt the prose

## Current Files

- `00_Bridge_Index.md` — doorway for bridge logic and how repeated details connect.
- `01_Act_Emotional_Map.md` — act-level emotional progression and key scene bridges.
- `03_Visual_Motif_Bridges.md` — visual motifs and how they change meaning across contexts.
- `05_Misdirection_Bridges.md` — misdirection, reflection, and perception-change notes.
- `90_Cosmology.md` — deep internal cosmology reference.
- `91_Unity_Explained.md` — Unity logic, perception, tragedy, and limits.
- `People_Before_System.md` — story principle for keeping characters before system explanation.
- `Luck_vs_Outcome.md` — internal rule for rare outcomes without turning them into fate.

## Rule

Reference files should make the story clearer for development, not harder to understand.

Ambiguity should come from story interpretation, not unclear documentation.

If a reference file repeats another file without adding a distinct function, merge it or remove it.
