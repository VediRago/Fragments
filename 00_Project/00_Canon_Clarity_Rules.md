# Canon Clarity Rules

## Purpose

Fragments should not contain lore that exists only because it sounds interesting.

Every important detail should explain what caused it, who it affects, and why it matters.

The goal is to avoid random lore, vague history, and unclear character logic.

## Main Rule

If a file says:

- a character becomes something
- a family line turns into something
- a system appears
- a belief spreads
- a tradition survives
- a law, institution, hierarchy, or custom exists
- a group accepts, rejects, fears, obeys, or depends on something

then the file must explain **why**.

## Required Chain

For character and background logic, use:

> experience -> emotion -> reason -> behavior

For place, culture, history, and institutions, expand it:

> experience -> emotion -> reason -> behavior -> atmosphere -> history

For game-facing ideas, use:

> world pressure -> character need -> obstacle -> approaches -> consequence

## No Ambiguous Compression

Avoid unsupported lines like:

- "He becomes ruthless."
- "The family turns to medicine."
- "This becomes law."
- "People accept the system."
- "The line becomes powerful."
- "The institution exists because of tradition."

These lines are only useful if the file also explains:

- what happened
- what the person or group felt
- what reason they formed
- why that reason caused the behavior
- who benefited
- who was hurt
- how the behavior survived long enough to become history

## Canon Note Rule

Project notes should be clearer than the in-world history.

Ambiguity can exist **inside the world** as disputed memory, propaganda, myth, or unreliable history.

Ambiguity should not exist in the project notes because the note forgot to explain the cause.

If something is uncertain, mark it clearly.

Use labels such as:

> Status: not locked yet.

or:

```md
<!-- FUTURE-ERA MATERIAL START
This material belongs to a later era and is kept here temporarily so the causal chain is not lost.
Do not treat it as part of this era's events.
FUTURE-ERA MATERIAL END -->
```

## Public And Private Truth Rule

When a file contains propaganda, public belief, hidden truth, or official history, separate them clearly.

Use headings or labels such as:

- Public story
- Private truth
- Political use
- Official claim
- Hidden cause

Do not mix what the public believes with what actually happened unless the file explicitly says it is doing that.

## Viriatus / Regulatus Clarity Rule

For the Founding Era, keep this locked:

- Viriatus's origin is hidden.
- House Ventari does not publicly reveal that origin.
- Publicly, Viriatus is framed as a child House Ventari trained, taught, disciplined, and shaped.
- House Ventari uses Viriatus as an asset for control.
- His victories become proof that Ventari discipline works.
- Old Rome's failures create public displeasure.
- House Ventari uses that displeasure to justify replacing the old order.
- Regulatus replaces the old order.
- House Ventari uses Regulatus to control the masses.
- House Ventari does not follow Regulatus the way ordinary citizens are expected to follow it.
- Viriatus does not create Regulatus.
- Ermerus turns Viriatus's death into Regulatus.

## Editing Rule

Before changing canon, check whether the change removes or damages the why.

Do not shorten a file by deleting the cause behind a behavior, institution, family wound, or world system.

Shorter is not better if it creates ambiguity.

A clean file is not the shortest file.

A clean file is the file where a new reader can understand what led to what.

## Review Questions

Before calling a file clear, ask:

- What pressure created this?
- Who experienced it?
- What emotion did it create?
- What reason did the person or group form?
- Why did that reason create this behavior?
- Who benefits from the behavior?
- Who is hurt by it?
- How does it survive into family, culture, institution, or history?
- If the information is uncertain, is it clearly marked as uncertain?
