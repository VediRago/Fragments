# Erit Worldbuilding

This folder documents the **Erit Worldbuilding** branch of *Fragments*.

This branch was built as a focused worldbuilding exercise over roughly **two days**, outside of full-time work. It is not presented as a finished game world. It is presented as evidence of process: how a single image in prose can expand into history, belief, institutions, family behavior, social pressure, and quest logic.

The starting image was simple:

> Two siblings play under the shadow of the emperor's statue.

The design question became:

> How can one historical wound become a civilization?

From that question, Erit's world expanded into Viriatus, House Ventari, Regulatus, dynastic family lines, medical dependency, identity chips, preservation culture, House Devaar, and the social pressure inherited by Erit, Sera, Kael, Fedra, and Valkare.

## What This Folder Demonstrates

This branch is meant to show more than lore creation.

It demonstrates:

- turning a prose detail into a world system
- building history from emotional cause rather than decoration
- connecting family behavior to public belief and institutional power
- creating ideology without making the society feel cartoonishly evil
- creating social systems that explain who benefits, who suffers, and why the system survives
- testing whether world pressure can become NPC behavior, quests, choices, and consequences

The core method is:

> experience -> emotion -> reason -> behavior

Repeated behavior can create atmosphere.

For world history, the chain can extend into:

> experience -> emotion -> reason -> behavior -> atmosphere -> history

For game-facing material, it can extend into:

> experience -> emotion -> reason -> behavior -> atmosphere -> quest logic

No important world detail should exist only because it sounds interesting.

Every system should answer:

- What experience or pressure created it?
- Who benefits from it?
- Who is harmed by it?
- Why do ordinary people accept it?
- How does it affect character behavior?
- How could it create quests, choices, or consequences?

## Core Worldbuilding Premise

Erit's world is not built around technology first.

It is built around grief, preservation, control, and inherited public belief.

Viriatus begins as a displaced child turned into a founder-symbol. House Ventari shapes his public meaning. His death is transformed into civic mythology. That mythology becomes Regulatus: a moral and social order that feels like duty, gratitude, family honor, and civilization to the people living under it.

Later, House Devaar rises through medicine, augmentation, preservation systems, treatment access, identity control, and bodily dependency.

The result is a society where control does not only come from violence. It comes from care, survival, medical need, family obligation, public gratitude, and fear of loss.

## Current Test Slice

The current playable / reviewable systems test for this branch is:

[Play the Layer System Test — Twine Prototype](https://vedirago.github.io/Fragments/Erit_Worldbuilding/Twine/Layer%20System/Index.html)

[View the Twine source file](Twine/Layer%20System/Index.html)

This prototype tests the Vys pressure system in Twine.

It uses three layers:

- Layer 1 — City / World State
- Layer 2 — Faction Pressure
- Layer 3 — NPC / Local Behavior

These labels belong to this first small test. The deeper idea is that layers represent different scales of experience, influence, and change.

The test checks whether faction actions, NPC behavior, city pressure, and future quest logic can influence each other without becoming simple reputation bars.

It is not a finished game section. It is a vertical slice for testing the method described in this folder.
