# Erit Worldbuilding Method

This folder keeps the method and rule files for the Erit Worldbuilding branch.

These files are not lore entries by themselves.

They explain how lore should be built so it does not become random background information.

## Rule

Worldbuilding should move from cause to consequence:

> experience -> emotion -> reason -> behavior -> atmosphere -> history

For playable or quest-facing material, the chain can extend to:

> experience -> emotion -> reason -> behavior -> atmosphere -> quest logic

## Files

- `Canon_Clarity_Rules.md` — rules for keeping project notes clear, so ambiguity comes from story interpretation rather than unclear documentation.
- `Worldbuilding_Method.md` — the broad method for building world pressure from story cause.
- `Method_Hierarchy.md` — how to decide what comes first and what should stay secondary.
- `Rules.md` — recurring design rules such as treasure map, key, and treasure.

## Constraint

A detail should not exist only because it sounds interesting.

It should connect to pressure, behavior, belief, system, conflict, perception change, or consequence.
