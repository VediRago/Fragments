# Erit Systems

This folder contains the working systems for Erit's world.

These files are not random mechanics or quest notes.

They show how world pressure can become:

- NPC behavior
- dialogue logic
- quest pressure
- player-facing conflict
- consequence

## Core Rule

Systems should follow the same clarity chain as the rest of Erit Worldbuilding:

> experience -> emotion -> reason -> behavior -> atmosphere -> quest logic

A system should not exist only because it sounds interesting.

It should explain what pressure created it, who benefits from it, who is hurt by it, why it survives, and how it can create choices or consequences.

## Files

- `NPC_Emotional_State_Model.md` — explains how NPC reactions come from layered pressure and emotional history.
- `NPC_System_Relationship_Model.md` — defines early working categories for how NPCs relate to Regulatus and other large systems.
- `Quest_Example_Chip_Removal.md` — shows how a quest seed can emerge from existing pressure around chips, medicine, identity, rebels, and control.
