# Prototype Noeme

Prototype Noeme is a focused visual storytelling test for adapting Fragments / Noeme into an image-led interactive reading experience.

This is not the full project.

The purpose is to test whether Fragments can work through:

- painterly visual style
- controlled panels
- disappearing / merging panel borders
- minimal text
- dialogue fragments
- motion through still images
- visual continuity anchors
- Twine-style pacing

## Core Idea

Noeme should feel like an illustrated narrative prototype, not a traditional game and not a normal visual novel.

The reader should not only read what happens. The reader should be guided through attention, movement, and recognition.

The visual grammar is:

```text
fragmented visual detail
-> guided attention
-> panel connection
-> full image / recognition
```

Panels are not just decoration. They represent fragmented perception. When panel borders disappear, fragments become continuity.

Later, during Unity, this can invert: a stable image can break back into fragments.

## Visual Style Direction

Use a consistent painterly style.

Preferred style:

```text
semi-realistic narrative painting, painterly brushwork, muted palette, cinematic lighting, emotional atmosphere, natural proportions, soft detail, restrained color, illustrated novel feeling, storyboard composition, not photorealistic, not anime, not comic book, not 3D render, not glossy concept art
```

The style should feel normal but slightly strange.

Do not make it flashy. No neon logo look. No exaggerated fantasy style. No heavy horror styling.

The goal is quiet unease and emotional focus.

## Production Order

1. Lock visual style.
2. Lock main characters.
3. Lock each world / setting.
4. Build panel tests.
5. Build complete prototype scenes.

Do not start by making random scenes.

## Asset Types

Use three types of images.

### 1. Scene Paintings

Full environments and establishing shots.

Examples:

- lake
- church
- lab
- Lucreta bedroom

Used for full reveals and backgrounds.

### 2. Focus Paintings

Close-ups that guide attention.

Examples:

- foot touching water
- ripple close-up
- hand on church door
- fingers sliding through wood
- tear falling
- flower crown
- grey eye

These can be separate images, not always crops from a full image.

### 3. Transition Paintings

Movement beats.

Examples:

- Kael lifting her arm
- Kael fist moving toward the floor
- door beginning to open
- shadow shortening in passing car light

These are used when motion itself carries the emotion.

## Panel System

Panels are used to guide attention.

Possible flow:

```text
panel 1 -> panel 2 -> panel 3 -> borders fade -> full image
```

Panels can show:

- movement
- emotional distance
- scene discovery
- focus narrowing
- continuity forming

Borders disappearing means the reader starts connecting fragments into one continuity.

During Unity, this can reverse: full image -> panels -> fragments -> color / sensation.

## Prototype Scenes

Prototype Noeme should contain complete scenes, not isolated screenshots.

Suggested prototype scenes:

1. NOEME title sequence
2. Lake scene
3. Church exit scene
4. Lucreta scene
5. Unity lab sequence through doctor / hospital transition

## NOEME Title Sequence

The title should use text, not image versions, so the letters can fade individually.

Sequence:

```text
NOEME
NEME
EME
ME
E
black
```

Letter disappearance order:

```text
O -> N -> E -> M -> E
```

Use a normal but slightly strange font. The font should not carry a heavy genre style.

Preferred feel:

- normal
- readable
- widely spaced
- soft off-white / ivory
- black background
- no flashy glow
- no neon

## Lake Scene Panel Idea

Use panels to guide attention toward water and ripples.

Possible sequence:

```text
Sera's foot lands in shallow rainwater
-> ripple forms
-> ripple expands
-> closer ripple
-> full lake image
```

The goal is not to show water. The goal is to teach attention.

Sera slowly disappears from the visual focus. The ripple becomes the memory anchor.

## Church Exit Scene Panel Idea

The scene assembles Phil and Abby through fragments.

Possible sequence:

```text
Phil's shoe steps onto church floor
-> Abby lifts her shoe / dress edge
-> hand reaches toward door
-> door opens slightly, showing shirt
-> door opens more, showing dress
-> doors open fully, revealing Phil and Abby outside
```

This is not just a church exit. It is a character reveal through fragments.

The reader assembles two people before seeing the full image.

## Lucreta Scene Panel Idea

Use close-ups to shift attention from touch to gaze.

Possible sequence:

```text
Lucreta's arm extended against the bed frame
-> fingers sliding through the creases / grooves of wood
-> fingers stop
-> head begins to lift
-> eyes begin to look upward
-> window / light / room reveal
```

The movement transfers from hand to head to gaze.

## Unity Lab Sequence

This is the most important sequence.

It is the breaking point of Kael's character.

The emotional progression:

```text
Sera
-> despair
-> Erit
-> decision
-> connection
-> Unity
```

Kael should first be localized in the lab.

Important early beats:

- Kael sitting against the mainframe
- arms around knees
- looking toward Sera's body
- tear falling
- arm lifting
- fist slamming
- head tilted back against the machine
- eyes closed
- looking toward connection port
- looking toward Erit
- fingers touching Erit's arm
- fingers sliding toward the cable / connector
- connection

The reader must feel Kael choosing Erit, not merely activating a machine.

Text should be minimal here. Images carry the emotion. Text survives mainly as broken attempts to speak:

```text
I-
I grr-
U.
```

## Unity Visual Rule

Only Unity should use POV fragments strongly.

Before Unity, the audience watches characters.

During Unity, the audience begins inhabiting fragments.

This makes POV itself a signal that something has changed.

## Unity Required Image Anchors

Unity must preserve the existing convergence sequence.

Do not replace, summarize, or reinterpret it casually.

Sequence:

```text
Bright sky
Mirror reflection
Crown of flowers
Bloody hand
Ripple in water
Light
Hands
Cars...
People...
Light
Black
White
Grey...
Hospital / doctor
```

References:

- Bright sky: shared image across major fragments
- Mirror: Kael
- Crown of flowers: Lucreta
- Bloody hand: Erit
- Ripple in water: Sera
- Light: ophthalmoscope
- Hands: doctor's hands
- Cars / People: hospital girl and Lucreta window bridge
- Black: Kael's eyes
- White: doctor's clothing / hospital light
- Grey: Kael's pale grey eyes -> hospital girl's eyes

## Technical Panel Test Needed

Before building full scenes, test in Twine:

1. Display three vertical panels.
2. Fade panel borders.
3. Reveal a full image.
4. Change images without using choice links.
5. Use timed animation for the NOEME title.

The project should feel like a prototype reel, not a full game.

## Key Principle

Noeme is not about making every scene beautiful.

It is about making the reader/viewer experience:

```text
fragment -> recognition -> continuity
```

If that works, the format works.
